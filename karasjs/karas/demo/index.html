1. Hello World! 基本的render方法接收2个参数：图表程序和渲染dom对象。
  2. csx csx取名借鉴自jsx，Canvas-Svg-Xml，根据根节点的不同，自动切换为canvas或svg类型，两者功能完全等价。
  3. 高清方案 在高清屏（如手机、mac）中，canvas会模糊或者1px边框问题，请自行调整meta元素的scale缩放（默认为1），或者为canvas设置style，使其宽高为属性的0.5倍（对应dpr为2的倒数）。
  4. dom标签 目前只有2种标签：div和span，分别对应block和inline元素。
  5. 弹性布局 flex基本被支持，这是在布局中很常见且实用的布局。
  6. 绝对定位 绝对定位也很重要，在一些提示窗口中有重大作用。
  7. 渐变 线性渐变和径向渐变可以丰富图形表现，css、canvas、svg写法定义均不相同，统一至css标准。
  8. 变形 变形转换提供了便利的几何变换，css、canvas、svg写法定义均不相同，统一至css标准。
  9. 事件支持 和jsx的事件标准一样，以on{Event}形式书写，所有事件被顶层canvas/svg劫持实现响应。
  10. 几何标签 几何标签规定以$开头，默认内置了几种基础几何形状：line/circle/rect/ellipse/sector/polyline/polygon。
  11. 自定义几何标签 如果觉得基本的几何图形不够用，或者为了复用代码，可以注册自定义几何标签。
  12. dom-diff 改变图形时需要重新渲染，对于canvas来说就是重绘，而svg重新生成的成本太大了，因此内置了dom-diff来提升性能。
  13. 响应事件 是否响应某个事件，内置已做好解析，确保事件发生的坐标在dom内且没有被遮盖，并考虑了matrix。如果想要自定义图形更加精准的响应，需要覆盖willResponseEvent方法。
  14. 可测试 ui测试是个非常麻烦的事情，这里提供了一种用virtual-dom来测试的方法。
  15. 图表示例 这里以常见的图表为例，展示了如何编写一个折线图，体现出代码量少、css规范零学习成本易维护的特点。
  16. 3d 结合自定义图形和线性渐变，以及结合线性渐变和transform，可以简单制作出较为炫的效果，比如3d柱状图。
  17. 火苗效果 结合图形标签和径向渐变，可以制作出火苗燃动的效果。
  18. 组件 组件可以极大地复用代码，同时它拥有setState改变数据的能力，另外贴合WebComponent标准，对事件、样式形成ShadowDom。
  19. 自定义事件 组件还拥有自定义事件功能，它以on-{event}形式书写，扩展了事件通信机制。
  20. icon 复杂的动态icon效果。
  21. css inline样式虽然可用但缺陷很多，传统css书写后为节点增加class的方式才是更好的方式。同样css也需要被预编译为cssx，传给根元素。
  22. component-css component中也可以为shadowRoot元素传入cssx，同时作为host的组件传入的cssx同样会覆盖shadowRoot的。
  23. 位图 除了矢量，还可以用img标签引用位图功能。
  24. 动画api 实现了Web Animation Api标准接口，并且增加了fps帧数控制。
  25. 动态加载 在一些展示或动画场景，希望能够通过配置的方式实现，这样能够灵活控制内容。动态加载能力实现了这一需求。
  26. 遮罩 可以为一个geom图形设置mask属性，这样它就作为下一个相邻节点的遮罩层出现。
